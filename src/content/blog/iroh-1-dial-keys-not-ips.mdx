Hace cuatro años un equipo pequeño empezó a construir una idea que suena a herejía: ¿y si las direcciones IP no fueran la forma correcta de identificar dispositivos en Internet? Hoy, con 8.800 estrellas en GitHub, 200 millones de endpoints creados en sus relays públicos solo en el último mes y una versión 1.0 recién publicada, Iroh demuestra que la herejía tenía sentido.

## El problema que Iroh resuelve

Las direcciones IP son frágiles por diseño. Cambian cuando te mueves de red, se rompen sin avisar, quedan atrapadas detrás de firewalls y NATs y no dicen nada sobre _quién_ está al otro lado. Llevamos décadas parchando esto con STUN, TURN, ICE, WebRTC, DNS dinámico… cada capa añade complejidad y puntos de fallo.

Iroh propone algo más radical: **usa claves criptográficas como direcciones**. En lugar de "conéctate a 192.168.1.42", dices "conéctate a _este_ dispositivo", identificado por su clave pública. El resto —encontrar la ruta, atravesar NATs, mantener la conexión viva— lo gestiona Iroh sin que tú muevas un dedo.

## Cómo funciona por dentro

Iroh se apoya en **QUIC** como transporte base, implementado a través de su propia stack ([noq](https://github.com/n0-computer/noq)). Esto le da cifrado extremo a extremo, streams concurrentes con prioridades, datagramas y ausencia de head-of-line blocking sin configuración adicional.

Sobre QUIC, Iroh añade tres capacidades que marcan la diferencia:

### 1. NAT traversal integrado

Iroh intenta siempre establecer una conexión directa mediante hole-punching. Si falla —porque ambos extremos están detrás de NATs simétricos, por ejemplo— recurre a relays públicos. El resultado: **el 95% de los datos transferidos viajan directamente entre dispositivos**, sin pasar por servidores intermedios.

### 2. QUIC multipath

Iroh implementa su propia capa de QUIC multipath. Puede establecer múltiples rutas dentro de una misma conexión (WiFi + 5G, por ejemplo) y hacer hot-swap si las condiciones de una ruta empeoran. La conexión no se entera.

### 3. Transportes personalizables

¿Quieres que Iroh funcione sobre Bluetooth Low-Energy? ¿Sobre LoRa? ¿Sobre WiFi Aware? ¿Sobre Tor? Iroh 1.0 expone un sistema de hooks y transportes custom que permite enchufar cualquier medio físico bajo la misma abstracción _dial-by-key_. La conexión se establece igual; el medio es transparente.

## Código: un echo server en Rust

El ejemplo canónico de Iroh es un servidor de echo. Así se ve el lado que acepta conexiones:

```rust
use iroh::{
    Endpoint,
    endpoint::{Connection, presets},
    protocol::{AcceptError, ProtocolHandler, Router},
};

const ALPN: &[u8] = b"iroh-example/echo/0";

#[derive(Debug, Clone)]
struct Echo;

impl ProtocolHandler for Echo {
    async fn accept(&self, connection: Connection) -> Result<(), AcceptError> {
        let endpoint_id = connection.remote_id();
        println!("conexión aceptada de {endpoint_id}");

        let (mut send, mut recv) = connection.accept_bi().await?;
        let bytes_sent = tokio::io::copy(&mut recv, &mut send).await?;
        println!("{bytes_sent} bytes copiados");

        send.finish()?;
        connection.closed().await;
        Ok(())
    }
}

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let endpoint = Endpoint::bind(presets::N0).await?;
    let router = Router::builder(endpoint)
        .accept(ALPN, Echo)
        .spawn()
        .await?;

    router.endpoint().online().await;
    println!("escuchando en: {}", router.endpoint().addr());

    // Aquí iría la lógica de conexión...
    router.shutdown().await?;
    Ok(())
}
```

Y el lado que conecta:

```rust
let endpoint = Endpoint::bind(presets::N0).await?;
let conn = endpoint.connect(addr, ALPN).await?;

let (mut send, mut recv) = conn.open_bi().await?;
send.write_all(b"Hello, world!").await?;
send.finish()?;

let response = recv.read_to_end(1000).await?;
assert_eq!(&response, b"Hello, world!");

conn.close(0u32.into(), b"bye!");
endpoint.close().await;
```

Fíjate en lo que **no** aparece: ni una IP, ni un puerto, ni configuración de NAT. `Endpoint::bind(presets::N0)` usa los relays públicos de n0 y el endpoint queda listo para recibir conexiones de cualquier parte del mundo. La dirección es un `EndpointAddr` que contiene la clave pública del endpoint.

## El mismo ejemplo en Python

Gracias a los bindings FFI generados con [uniffi-rs](https://github.com/mozilla/uniffi-rs), Iroh 1.0 funciona igual de limpio desde Python:

```python
import asyncio
import iroh

ALPN = b"iroh-ffi/example/0"

async def accept():
    opts = iroh.EndpointOptions(alpns=[list(ALPN)])
    ep = await iroh.Endpoint.bind(opts)

    print("escuchando en:", ep.id())
    ticket = iroh.EndpointTicket.from_addr(ep.addr())
    print("ticket:", str(ticket))

    incoming = await ep.accept_next()
    accepting = await incoming.accept()
    conn = await accepting.connect()

    bi = await conn.accept_bi()
    data = await bi.recv().read_to_end(1024)
    print("recibido:", data.decode())

    await bi.send().write_all(data)
    await bi.send().finish()
    await ep.close()

async def connect(ticket_str):
    ticket = iroh.EndpointTicket.from_string(ticket_str)
    addr = ticket.endpoint_addr()

    ep = await iroh.Endpoint.bind(iroh.EndpointOptions())
    conn = await ep.connect(addr, list(ALPN))

    bi = await conn.open_bi()
    await bi.send().write_all(list(b"hello"))
    await bi.send().finish()

    echoed = await bi.recv().read_to_end(1024)
    print("respuesta:", echoed.decode())
    await ep.close()
```

El patrón es idéntico: `Endpoint.bind()`, `connect()` con una dirección-opaca-que-es-una-clave, streams bidireccionales. La API de Python expone exactamente los mismos conceptos que la de Rust. Y lo mismo vale para Node.js, Kotlin y Swift.

## Los protocolos incorporados

Iroh no es solo un tubo. Sobre la capa de conexión, el ecosistema incluye tres protocolos listos para usar:

- **iroh-blobs**: transferencia de archivos binarios direccionados por contenido (hash BLAKE3). Escala de kilobytes a terabytes sin cambiar de API.
- **iroh-gossip**: redes publish-subscribe que un teléfono medio puede manejar sin sudar. Útil para chat, colaboración en tiempo real, sincronización de estado.
- **iroh-docs**: almacén clave-valor con replicación libre de conflictos (CRDT) sobre iroh-blobs. Piensa en un Google Docs descentralizado como primitiva de librería.

La gracia está en que son modulares: usas solo lo que necesitas. ¿Solo quieres enviar streams de vídeo? Usas la capa base. ¿Necesitas sincronizar documentos entre peers? Añades iroh-docs. ¿Quieres descubrir peers por tema? Activas iroh-gossip.

## Compatibilidad real

La versión 1.0 trae bindings oficiales para **Python, Node.js, Kotlin y Swift**, además del crate Rust nativo. Esto significa que puedes empotrar Iroh en una app iOS en Swift, en una app Android en Kotlin, en un backend en Python y en un frontend web (compila a WASM) y todos se hablan entre sí con el mismo protocolo.

El equipo garantiza **estabilidad de wire protocol** para toda la major version 1: un endpoint Iroh v1 se comunicará con otro endpoint Iroh v1 independientemente de la versión menor o del lenguaje en que esté escrito.

## Mi veredicto

Iroh 1.0 me parece una de esas piezas de infraestructura que tardas cinco minutos en probar y dos horas en darte cuenta de todas las implicaciones. La abstracción _dial-by-key_ es tan natural que cuesta creer que no fuera el estándar desde el principio.

Dicho esto, Iroh no es para todo. Si tu caso de uso es un servidor HTTP tradicional sirviendo a miles de clientes efímeros, no necesitas P2P. Pero si estás construyendo algo que implica dispositivos que se mueven, colaboración offline-first, sincronización entre pares, IoT sin cloud, o simplemente quieres conexiones que no se rompan cuando el usuario cambia de WiFi a 5G, Iroh resuelve problemas que antes requerían meses de ingeniería.

El hecho de que los relays públicos hayan visto 200 millones de endpoints en 30 días sugiere que ya hay bastante gente usándolo en producción. Con la estabilidad de la 1.0 y los bindings multiplataforma, la barrera de entrada acaba de bajar al suelo.

Si te pica la curiosidad, el quickstart son cinco minutos: clonas el repo, ejecutas `cargo run --example quickstart receiver` en un terminal y `cargo run --example quickstart sender <ticket>` en otro. Ves el round-trip time aparecer y entiendes que acabas de establecer una conexión cifrada, directa, sin configurar una IP, un puerto ni un firewall.

Eso es Iroh. Y acaba de cumplir la mayoría de edad.
