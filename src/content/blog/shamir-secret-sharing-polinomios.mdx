En 1979, Adi Shamir —la "S" de RSA— publicó un artículo de apenas dos páginas titulado _How to Share a Secret_. La idea era tan elegante como práctica: dividir un secreto en _n_ trozos de forma que necesites al menos _k_ de ellos para reconstruirlo. Con _k-1_ trozos no revelas absolutamente nada. No "es difícil de crackear". **Nada**. Cero bits de información.

Cuarenta y siete años después, el algoritmo sigue siendo la base de sistemas como HashiCorp Vault, los kits de recuperación de Ente, carteras multisig de Bitcoin y cualquier sitio donde necesites que varias personas autoricen algo sin que ninguna tenga la llave completa.

## Dos puntos hacen una recta

El punto de partida es algo que aprendiste en el instituto: dos puntos determinan una única recta. Un solo punto no —infinitas rectas pasan por él.

Ahora esconde el secreto donde la recta cruza el eje vertical. Imagina que el secreto es el número 7. Dibujas una recta aleatoria que cruce el eje Y justo en 7. La pendiente no importa —es ruido que oculta el secreto.

A cada persona le das un punto de la recta. Con un solo punto, cualquier secreto es posible. Con dos puntos, la recta queda fijada y el secreto se revela.

Esto ya es un esquema 2-de-_n_. Puedes generar tantos puntos como quieras y cualquier par reconstruye el secreto.

## Más curvas, más seguridad

Si quieres que hagan falta 3 personas, usas una parábola. Para 4, una cúbica. En general, para un umbral _k_ necesitas un polinomio de grado _k-1_.

El secreto sigue siendo el valor del polinomio en _x=0_. Los demás coeficientes son aleatorios. Cada _share_ es un punto _(x, P(x))_.

La propiedad que lo hace seguro es la **seguridad perfecta** (_information-theoretic security_): con _k-1_ shares, todos los valores posibles del secreto siguen siendo igual de probables. No es que sea computacionalmente difícil de romper —es que literalmente no hay información.

## Vamos al código

Suficiente teoría. Implementemos Shamir desde cero en Python.

Primero, la generación de shares. Necesitamos aritmética sobre un campo finito para garantizar la seguridad perfecta —si usáramos floats, los errores de redondeo filtrarían información. Usaremos un primo grande como módulo.

```python
import secrets
from dataclasses import dataclass

# Primo de 256 bits para el campo finito
PRIME = 2**256 - 2**32 - 2**9 - 2**8 - 2**7 - 2**6 - 2**4 - 1

@dataclass(frozen=True)
class Share:
    x: int
    y: int


def generate_shares(secret: int, threshold: int, total: int) -> list[Share]:
    """Divide un secreto en 'total' shares, recuperable con 'threshold'."""
    if threshold > total:
        raise ValueError("El umbral no puede superar el total de shares")
    if threshold < 2:
        raise ValueError("El umbral mínimo es 2")

    # Coeficientes aleatorios para el polinomio de grado threshold-1
    coeffs = [secret] + [secrets.randbelow(PRIME) for _ in range(threshold - 1)]

    shares = []
    for x in range(1, total + 1):
        y = _evaluate_polynomial(coeffs, x)
        shares.append(Share(x=x, y=y))

    return shares


def _evaluate_polynomial(coeffs: list[int], x: int) -> int:
    """Evalúa un polinomio en x usando el método de Horner."""
    result = 0
    for coeff in reversed(coeffs):
        result = (result * x + coeff) % PRIME
    return result
```

Para reconstruir el secreto usamos **interpolación de Lagrange**, que calcula el polinomio que pasa por _k_ puntos sin necesidad de resolver sistemas de ecuaciones:

```python
def recover_secret(shares: list[Share]) -> int:
    """Reconstruye el secreto a partir de k shares usando interpolación de Lagrange."""
    if len(shares) < 2:
        raise ValueError("Se necesitan al menos 2 shares")

    secret = 0
    for i, si in enumerate(shares):
        # Numerador y denominador del coeficiente de Lagrange para P(0)
        num = 1
        den = 1
        for j, sj in enumerate(shares):
            if i == j:
                continue
            # P(0) → el numerador usa -sj.x, el denominador usa (si.x - sj.x)
            num = (num * (-sj.x)) % PRIME
            den = (den * (si.x - sj.x)) % PRIME

        # Multiplicar por el inverso modular del denominador
        lagrange_coeff = (num * pow(den, -1, PRIME)) % PRIME
        secret = (secret + si.y * lagrange_coeff) % PRIME

    return secret
```

Y aquí la prueba de que funciona:

```python
# Dividimos el número 42 en 5 shares, recuperable con 3
shares = generate_shares(secret=42, threshold=3, total=5)

print(f"Shares generadas: {len(shares)}")
# Recuperamos con las 3 primeras
recovered = recover_secret(shares[:3])
print(f"Secreto recuperado: {recovered}")  # 42

# Con solo 2 shares no debería funcionar
wrong = recover_secret(shares[:2])
print(f"Con 2 shares: {wrong}")  # ≠ 42 — cualquier valor, pero no el secreto
```

El truco está en `pow(den, -1, PRIME)` —Python 3.8+ soporta exponentes negativos en `pow()` para calcular el inverso modular, que es lo que necesitamos para dividir en un campo finito.

## Por qué se usa en el mundo real

Un par de ejemplos concretos donde esto no es teoría:

- **Ente** lo usa en su _Legacy Kit_: divides la clave de recuperación de tu cuenta entre varias tarjetas físicas. Ninguna tarjeta contiene la clave por sí sola. Si pierdes una, las revocas y generas otras nuevas sin cambiar la clave maestra.
- **HashiCorp Vault** implementa Shamir para desprecintar (_unseal_) el almacén. Necesitas que _k_ administradores introduzcan su parte para arrancar Vault tras un reinicio.
- **Carteras multisig** de Bitcoin usan un concepto relacionado (aunque técnicamente no es Shamir puro —SLIP-39 sí lo es).

## Mi opinión

Lo que más me flipa de Shamir's Secret Sharing no es la criptografía en sí, sino la elegancia de la idea. En un sector donde la seguridad suele ser "añade más capas y reza", esto es lo contrario: un esquema matemáticamente demostrable, con cero suposiciones computacionales, que puedes explicar en una servilleta.

El algoritmo tiene 47 años. En ese tiempo hemos visto caer MD5, SHA-1, RC4 y mil cosas más. El esquema de Shamir sigue intacto porque no depende de que factorizar sea difícil ni de que nadie tenga un ordenador cuántico —es seguridad perfecta en el sentido de Shannon.

Dicho esto, implementarlo bien no es trivial. La documentación técnica de Ente sobre su Legacy Kit menciona varios detalles que se te escapan en una primera versión: qué primo usar, cómo manejar secrets que no son números, cómo revocar shares sin reemitir todo. Pero el núcleo son 30 líneas de Python. Y eso es lo que lo hace brillante.
